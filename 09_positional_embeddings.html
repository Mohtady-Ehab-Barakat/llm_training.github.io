Positional embeddings

Sinusoidal · Absolute learned · Relative · RoPE — why position matters and how each method works

Why position is needed at all

Attention computes dot products between Q and K vectors derived from token embeddings. Those embeddings encode meaning — "cat" always produces the same vector regardless of where it appears in the sentence. So the dot products are completely blind to order. "The cat sat" and "sat cat The" produce identical attention patterns. For language this is catastrophic — "dog bites man" and "man bites dog" must mean different things. Position information must be injected from outside.

The problem — attention is permutation-invariant
Shuffle the tokens — the per-token attention patterns are identical regardless of order, because Q and K only encode meaning, not position.

1 — Sinusoidal embeddings (original transformer, 2017)

The simplest approach: create a unique D-dimensional vector for every position using sine and cosine waves, then add it to the token embedding before anything else:

PE(pos, dim) = sin(pos / 10000^(dim/D))   # even dimensions
             = cos(pos / 10000^(dim/D))   # odd dimensions

Think of it like a clock with many hands. Dimension 0–1 is a seconds hand — it completes a full cycle very quickly and distinguishes adjacent positions. Dimension 30–31 is an hour hand — it moves slowly and distinguishes distant positions. Together all D/2 hands produce a unique fingerprint for every position up to ~10,000 tokens. No parameters to learn — pure math.

2 — Absolute learned embeddings (GPT-2, GPT-3)

Instead of a fixed formula, make the position vectors learned parameters — a table of size [T_max, D] trained by gradient descent:

x = token_embedding[token_id] + position_embedding[pos]
# position_embedding is a [T_max, D] weight matrix
# learned just like any other parameter

Simpler to implement, and the model learns whatever position representation is most useful for its data. The weakness is sharp: positions beyond T_max were never seen during training, so the model completely breaks at those lengths. Also uses T_max × D extra parameters. GPT-2 used T_max = 1024; GPT-3 used 2048.

Absolute vs relative: both sinusoidal and learned absolute embeddings tell the model "you are at position 5." What attention really cares about is "this token is 3 steps left of that one." Relative and RoPE methods encode distance directly, which generalises better.

3 — Relative position embeddings (T5, ALiBi)

The core insight: what attention cares about is not "I am at position 5" but "this token is 3 steps away from that one." Relative embeddings modify the attention score directly. Instead of:

score(i, j) = Qᵢ · Kⱼ                     # plain attention

You add a bias that depends on the distance between positions:

score(i, j) = Qᵢ · Kⱼ + bias(i − j)      # relative attention

# T5 style: bias is a learned scalar per distance bucket
# ALiBi style: bias = −|i−j| × slope  (no learned params)
#              slope is fixed per attention head

The model learns "tokens 3 apart should attend this much" independently of their absolute positions. This generalises better across lengths. ALiBi (Attention with Linear Biases) uses a fixed linear penalty — no extra parameters at all — and was shown to extrapolate to sequences much longer than those seen in training.

4 — RoPE: Rotary Position Embeddings (LLaMA, Mistral, Gemma)

RoPE is the most elegant solution. Instead of adding position information to embeddings, it rotates the Q and K vectors by an angle proportional to the position, inside the attention computation. For each consecutive pair of dimensions:

rotate(v, pos, i) = [v₀·cos(pos·θᵢ) − v₁·sin(pos·θᵢ),
                     v₀·sin(pos·θᵢ) + v₁·cos(pos·θᵢ)]

where θᵢ = 10000^(−2i/D)   # same frequencies as sinusoidal

Q gets rotated by m×θ (its position). K gets rotated by n×θ (its position). Then the dot product is computed normally. The magic: the dot product of two rotated vectors depends only on the difference (m−n), not the absolute positions — a mathematical property of how rotations compose.
