Activations & Gating

ReLU · GeLU · Swish · ELU · SeLU · GLU · ReGLU · GeGLU · SwiGLU · LiGLU

What is an activation function?

A neural network without activation functions is just a chain of matrix multiplications. No matter how many layers you stack, the whole thing collapses into one single linear transformation — it can only fit straight lines through data, which can't learn anything interesting.

An activation function is applied to the output of each layer to introduce a bend. Once you can bend, you can fit any shape. Think of it like this: a straight wire can only make a flat form. Add bends and you can make any sculpture. Activation functions are the bends.

The core question every activation function answers: given a number x coming out of a neuron, what number should I pass forward?

What is gating?

Simple activations apply the same fixed rule to every number. Gating is more powerful: let the network learn what to block and what to pass through, dynamically, based on the input itself.

In a gated unit, the input is used twice — once to produce a "content" signal and once to produce a "gate" signal (a number near 0 or 1). The final output is content × gate. When gate ≈ 1, content flows through freely. When gate ≈ 0, content is blocked. Both the content projection and the gate projection use separate learned weight matrices, so the network learns which patterns to let through.

Analogy: a valve on a pipe. The content is the water. The gate is the valve position. But unlike a manual valve, the valve position is computed from the water pressure itself.

Simple activations — explained plainly

ReLU is the most famous. The rule is brutal and simple: if x is negative, output 0. If positive, output x unchanged. Like a one-way valve — negative signals are completely blocked. Fast and easy. The problem: once a neuron outputs 0, its gradient is also 0 and it can never update again — a "dead neuron." In a big network, many neurons can die permanently.

ELU fixes dead neurons by giving the negative side a smooth exponential curve instead of flat zero. Negative inputs produce small negative outputs, so the gradient is never exactly zero and neurons can always recover. The parameter α (usually 1.0) controls how far negative the output goes.

SeLU is ELU with two specific magic constants (λ ≈ 1.0507, α ≈ 1.6733) derived mathematically. These exact values make the network self-normalizing — stack many SeLU layers and activations automatically stay near mean=0, std=1 without any LayerNorm. Clever, but rare in transformers since we do normalisation explicitly.

GeLU is what BERT, GPT-2, and GPT-3 use. Instead of a hard cut at zero, the gate is probabilistic. A large positive is almost certainly passed through. A large negative is almost certainly blocked. Numbers near zero are passed partially — a small positive might be 60% through, a tiny negative might be 40% blocked. The curve looks like ReLU with a gentle S-shape around zero. Works better than ReLU for language models.

Swish (also called SiLU) multiplies x by sigmoid(x). Sigmoid is always between 0 and 1, so it acts as a soft gate on the signal itself. The interesting feature is a small dip below zero for slightly negative inputs — the curve is non-monotone, which helps the network learn richer patterns. This is the base function inside SwiGLU.

Gated units — explained plainly

GLU (Gated Linear Unit) is the parent of the whole gating family. The input x is linearly projected into two separate vectors using two different weight matrices W and V. One gives the content, the other goes through sigmoid (→ 0 to 1 range) to give the gate. Output = content × gate. The network learns both W and V during training.

ReGLU replaces sigmoid in the gate with ReLU. The gate is now either zero (fully blocked) or positive (open). Simpler and faster — no exp() inside the sigmoid. Less smooth but still effective.

GeGLU replaces sigmoid with GeLU in the gate. The gate is smooth and probabilistic — partially-good signals can partially pass through. Softer and more expressive than ReGLU.

SwiGLU replaces sigmoid with Swish in the gate. This is the one that matters most — it's what LLaMA, Mistral, and Gemma use in every FFN layer. It empirically outperforms GeLU and all other variants on language modeling benchmarks. Nobody has a perfect theoretical explanation for why — it just works. One practical note: because SwiGLU uses two projections instead of one, the hidden FFN size changes from 4D to 8D/3 to keep parameter count equal.

LiGLU removes the activation function from the gate entirely — two linear projections multiplied together, no nonlinearity at all. Surprisingly competitive in research. Very fast since there's no exp() or erf() to compute.

The family tree

── Simple activations ──────────────────────────────────── ReLU → max(0, x) hard cut, fast, dead neurons ELU → x if x>0, α(eˣ−1) if x≤0 smooth negatives, no dead neurons SeLU → ELU with magic constants self-normalising, rare in LLMs GeLU → x · Φ(x) smooth probabilistic gate ← GPT-2/3 Swish → x · σ(x) smooth, non-monotone ← base of SwiGLU ── Gating family (all use two weight matrices: W and V) ── GLU → (xW) ⊙ σ(xV) parent — sigmoid gate ReGLU → (xW) ⊙ ReLU(xV) hard gate GeGLU → (xW) ⊙ GeLU(xV) smooth probabilistic gate SwiGLU → (xW) ⊙ Swish(xV) smooth learned gate ← LLaMA, Mistral, Gemma LiGLU → (xW) ⊙ (xV) no nonlinearity in gate
What to remember: for modern LLMs the answer is almost always SwiGLU in the FFN layers. If you open a LLaMA or Mistral config file you'll see hidden_act: "silu" — that's Swish/SiLU, the gate activation inside SwiGLU. The FFN hidden size will be 8/3 × model_dim rather than the 4× you'd expect from a plain FFN, because the gating unit uses two projections.
Dead neuron
A neuron stuck at output=0 forever because ReLU zeroed it and gradient can't update it. Fixed by ELU, GeLU, Swish.
Monotone
Always increasing or always decreasing. ReLU, ELU, GeLU are monotone. Swish is not — it dips slightly below zero for small negatives.
Smooth
No sharp corners or kinks. GeLU, Swish, ELU are smooth. ReLU has a kink at x=0. Smooth → better gradient flow.
Gating
Using one signal to control how much of another signal passes through. The gate is learned — the network decides what to let through.
⊙
Element-wise multiplication (Hadamard product). Two vectors of the same shape, multiplied position by position.
SiLU
Sigmoid Linear Unit — the same function as Swish. x · σ(x). When you see "silu" in model configs, it means Swish, which means SwiGLU in the FFN.
