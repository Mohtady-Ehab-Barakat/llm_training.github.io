Weight decay

What it is · why it works · weight decay vs L2 regularisation · the Adam/AdamW difference · which params to decay

What weight decay is

Every step of gradient descent updates a weight like this:

w ← w − lr × gradient                    # plain SGD

Weight decay adds one extra operation — multiply the weight by a number slightly less than 1 before applying the gradient:

w ← w × (1 − lr·λ) − lr × gradient
      ↑ this is weight decay — w shrinks every step

The factor (1 − lr·λ) is always less than 1. So every single training step, every decayed weight shrinks by a tiny fraction. If the task gradient keeps pushing a weight up, the weight finds an equilibrium. If the gradient stops pushing, the weight shrinks toward zero. Weights that earn their place stay. Weights that don't gradually disappear.

Why weight decay prevents overfitting

When a model overfits it typically develops very large weights that carve out sharp, complex decision boundaries tuned to the noise in training data. Large weights produce large activations, overconfident predictions that are right on training data but wrong on new data.

Weight decay prevents this by taxing large weights. A weight of 10 decays 10× faster in absolute terms than a weight of 1. The model always pays a cost for having large weights, so it only keeps them large when the task data strongly demands it. The result is smoother, more generalisable solutions.

Weight decay vs L2 regularisation — not the same with Adam

Both methods try to keep weights small but do it differently.

L2 regularisation adds a penalty term to the loss:

Loss_total = Loss_task + λ · Σ w²
# gradient of penalty = 2λw → added to task gradient before update

Weight decay multiplies the weight directly, after the optimiser step:

w ← w × (1 − lr·λ) − lr × optimiser_update(gradient)

With SGD they are mathematically identical. Substituting 2λw into the SGD update gives exactly w × (1 − 2·lr·λ) — same formula. This is why the terms were used interchangeably for years.

With Adam they are completely different. Adam divides every gradient by an adaptive denominator √v̂ based on gradient history. When you add λw to the gradient as L2, it also gets divided by √v̂:

# Adam + L2:
w ← w − lr × (task_grad + λw) / √v̂
                         ↑ penalty is scaled down by √v̂

# The effective regularisation per weight depends on its gradient
# history — inconsistent, and empirically worse.

AdamW fixes this by keeping weight decay completely separate from the gradient:

# AdamW:
w ← w × (1 − lr·λ) − lr × task_grad / √v̂
      ↑ decay applied directly to w, not via the gradient pipeline

# Every weight gets the same proportional decay regardless of
# its gradient history. Cleaner and consistently better.
This is exactly why AdamW exists (Loshchilov & Hutter, 2017). With SGD the difference doesn't matter — use either. With Adam always use AdamW. Every modern LLM (GPT-3, LLaMA, Mistral, Gemma) trains with AdamW, not Adam.

Which parameters get decayed

Not everything should be decayed. The standard rule: decay weight matrices, leave normalisation parameters and biases alone.

# PyTorch implementation — two parameter groups:
decay_params    = [p for n,p in model.named_parameters()
                   if p.dim() >= 2]          # matrices
no_decay_params = [p for n,p in model.named_parameters()
                   if p.dim() < 2]           # norms, biases

optimizer = AdamW([
    {'params': decay_params,    'weight_decay': 0.1},
    {'params': no_decay_params, 'weight_decay': 0.0},
], lr=3e-4, betas=(0.9, 0.95))

The equilibrium — where weights settle

At convergence each weight finds an equilibrium between two opposing forces: the task gradient pushing it toward values that reduce the loss, and weight decay pulling it toward zero:

# At equilibrium (decay force ≈ task gradient):
lr·λ·w ≈ task_gradient
→ w ≈ task_gradient / (lr·λ)

# A weight that contributes a lot to reducing the loss has a
# large task gradient → settles at a large value.
# A weight that contributes little → settles near zero.

Weight decay is a continuous, differentiable way of saying: "prove you're useful or disappear."

Typical values

λ = 0.1
Near-universal standard for large transformers. GPT-3, LLaMA 1/2/3, Mistral, Gemma all use 0.1.
λ = 0.01
Smaller models, fine-tuning runs, or when the dataset is small and you want gentler regularisation.
λ = 0.0
No weight decay. Occasionally used for very large datasets where overfitting isn't a concern.
λ > 0.1
Rarely used — too aggressive. Forces weights too small, hurting model capacity.
Decay factor/step
(1 − lr·λ) per step. With lr=0.001 and λ=0.1: factor = 0.9999. Weight halves every ~6,931 steps.
dim >= 2 rule
PyTorch convention for separating matrices (decay) from 1D params like norms and biases (no decay). Clean and reliable.
Summary: weight decay = multiply every weight matrix by (1 − lr·λ) every step. With SGD it equals L2 regularisation. With Adam it doesn't — use AdamW which applies decay directly to weights, bypassing the adaptive scaling. Standard λ = 0.1. Do not decay LayerNorm parameters or biases.
