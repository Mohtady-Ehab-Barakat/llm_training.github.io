Training dynamics

Optimizers

Momentum · Adagrad · RMSProp · Adam · AdamW — what each one fixes

During training, the model's parameters (weights) need to be nudged after every batch to reduce the loss. The gradient tells you which direction reduces the loss. An optimizer decides how much to nudge each weight, and in what way, given that gradient. The simplest version — vanilla gradient descent — is just w = w − lr × gradient. Every optimizer below is an improvement on that idea, each one fixing a specific weakness of the previous.

What is an optimizer?

After computing the loss and running backpropagation to get gradients, the optimizer applies an update rule to each weight. The update rule can be as simple as subtracting a fixed fraction of the gradient, or as sophisticated as maintaining per-weight statistics that adapt the step size dynamically.

The key things an optimizer controls are: how fast to move (learning rate), whether to smooth over noisy gradients (momentum), and whether to slow down for weights that already get large updates (adaptive learning rates).

The optimizers — in order of evolution

Momentum
Fixes: oscillation and slow convergence of vanilla SGD
v = β·v + lr·∇L # build up velocity in gradient direction w = w − v # move along velocity, not raw gradient
lr ≈ 0.01β = 0.9

Vanilla gradient descent reacts only to the current gradient — in a narrow valley this causes it to bounce off the steep walls repeatedly. Momentum fixes this by accumulating a velocity vector. Past gradients keep pushing you in the same direction, so you accelerate along consistent directions (the valley floor) and dampen oscillations across steep ones. β = 0.9 means 90% of the old velocity survives each step. You can think of it as a ball rolling downhill: it doesn't stop instantly when the slope changes.

Adagrad
Fixes: same learning rate for all weights regardless of gradient frequency
G = G + (∇L)² # sum of squared gradients — grows forever w = w − (lr / √(G+ε)) · ∇L
lr ≈ 0.01ε = 1e-8

Different weights have very different gradient magnitudes. Common-feature weights get large gradients often and would overshoot with a fixed lr. Rare-feature weights barely update. Adagrad gives each weight its own effective learning rate by dividing by the square root of the cumulative sum of squared gradients. Weights that have seen large gradients get smaller steps; weights that rarely update keep larger steps.

Adagrad's problem: G only ever grows, so the effective learning rate shrinks toward zero and training eventually stalls completely. This is why you see it slow dramatically in the animation.
RMSProp
Fixes: Adagrad's vanishing learning rate — uses exponential moving average instead
E = β·E + (1−β)·(∇L)² # exponential moving avg — forgets old history w = w − (lr / √(E+ε)) · ∇L
lr ≈ 0.001β = 0.9ε = 1e-8

RMSProp replaces Adagrad's forever-growing sum with an exponential moving average of squared gradients. Old gradient history decays away, so G reflects recent gradient magnitudes, not all-time ones. The effective learning rate stabilises rather than vanishing. Proposed by Geoff Hinton in a Coursera lecture — famously one of the most-cited unpublished works in deep learning.

Adam — Adaptive Moment Estimation
Fixes: combines momentum + RMSProp, adds bias correction for initialisation
m = β₁·m + (1−β₁)·∇L # 1st moment: mean of gradients (momentum) v = β₂·v + (1−β₂)·(∇L)² # 2nd moment: mean of sq. gradients (RMSProp) m̂ = m / (1 − β₁ᵗ) # bias correction (t = step number) v̂ = v / (1 − β₂ᵗ) # bias correction w = w − lr · m̂ / (√v̂ + ε)
lr ≈ 3e-4β₁ = 0.9β₂ = 0.999ε = 1e-8

Adam maintains two running averages: m (first moment — mean of gradients, like Momentum's velocity) and v (second moment — mean of squared gradients, like RMSProp's E). It also adds bias correction: both m and v start at zero, which biases early estimates downward. Dividing by (1 − β^t) corrects for this. The result is fast, stable, per-parameter adaptive convergence — the default optimizer for virtually all LLM training.

AdamW
Fixes: Adam's weight decay is distorted by the adaptive learning rate — decouples it
# same m, v, m̂, v̂ update as Adam, then: w = w·(1 − lr·λ) − lr · m̂ / (√v̂ + ε) # ↑ weight decay applied directly to w, not via the gradient
lr ≈ 3e-4β₁ = 0.9β₂ = 0.999λ = 0.1

In standard Adam, adding L2 regularisation means the regularisation term gets scaled by the adaptive learning rate 1/√v̂ — making the effective weight decay vary unpredictably per weight. AdamW decouples weight decay by applying it directly to the weights before the gradient step, so regularisation behaves uniformly regardless of each weight's gradient history. AdamW is the standard optimizer for transformer training (GPT, LLaMA, Mistral etc.).

The progression at a glance

SGD
Baseline. Subtract lr × gradient. Simple but sensitive to lr choice and oscillates in narrow valleys.
+ Momentum
Add velocity accumulation. Smooths oscillation, accelerates in consistent directions.
+ Adagrad
Per-weight adaptive lr scaled by cumulative gradient history. Lr eventually dies.
+ RMSProp
Replace cumulative sum with exponential moving average. Lr stabilises.
= Adam
Momentum + RMSProp + bias correction. Fast, robust, near-universal default.
→ AdamW
Adam with decoupled weight decay. Better regularisation, used for all modern LLMs.
In practice: AdamW with a cosine learning rate schedule (warm-up for ~1–2% of steps, then decay to ~10% of peak lr) is the recipe used for GPT-3, LLaMA, Mistral, and most other large models. The typical hyperparameters are β₁ = 0.9, β₂ = 0.95 (slightly lower than Adam's default 0.999 for stability at large batch sizes), and weight decay λ = 0.1.
