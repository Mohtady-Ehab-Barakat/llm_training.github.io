Residual connections

The simplest idea that made deep networks possible — and why transformers are built around it

What is a residual connection?

In a normal layer, you compute an output and it replaces the input entirely:

x = F(x)       # plain — new x completely replaces old x

With a residual connection, you compute a correction and add it to the original:

x = x + F(x)   # residual — original x is preserved and enriched

That's the entire idea. The sub-layer F is now only responsible for computing the difference — the correction — not the full output. The original signal flows straight through on what's called the residual highway or identity shortcut.

Why training without residuals fails

Imagine playing telephone with 50 people. Each person slightly mishears and rephrases the message. By person 50, the original is unrecognisable. The same happens to signals in a deep network — each layer transforms the signal and after many layers the original information is mangled.

The bigger problem is the backward pass. Gradients — the error signals telling each layer how to improve — flow backward from the loss through every layer. At each layer the gradient is multiplied by that layer's local derivative. If those derivatives are less than 1 (which they often are), the gradient shrinks. After 50 multiplications of 0.9, you have 0.9⁵⁰ ≈ 0.005 — essentially zero. Early layers see no useful signal and never learn. This is the vanishing gradient problem, and it's why networks deeper than ~20 layers were impossible to train before 2015.

How residuals solve it — the math

The derivative of a sum is the sum of derivatives. So for x_out = x_in + F(x_in):

d(loss)/d(x_in) = d(loss)/d(x_out) · (1 + dF/dx_in)

See the 1 in (1 + dF/dx_in)? Even if dF/dx_in shrinks to nearly zero, the gradient still flows back with full strength through the residual path. The gradient doesn't fight through every layer — it rides the highway straight back to layer 1.

The learning target changes: without residuals, each layer must learn to map x → y where y could be anything. With residuals, each layer only needs to learn F(x) = y − x — just the small correction. Starting from F(x) = 0 (tiny weights) is always valid, and the network starts near the identity function. This is why residual networks can be initialised and trained stably from scratch.

Residuals in transformers

In a transformer, every single sub-layer — both attention and FFN — is wrapped in a residual connection. This is not optional; it's the backbone of the whole architecture:

# One transformer block, pre-norm style:
x = x + Attention(LayerNorm(x))   # residual around attention
x = x + FFN(LayerNorm(x))         # residual around FFN

The x flowing through these additions is called the residual stream. Think of it as a shared whiteboard that every layer reads from and writes to. Each layer adds new information but never erases old information — the whiteboard only grows richer.

In a 96-layer model like GPT-3, layer 1 and layer 96 are only ever one addition away from each other in the gradient graph. The gradient from the loss reaches layer 1 almost as strongly as it reaches layer 96. All 96 layers train simultaneously and equally.

Interpretability angle: because each layer only adds to the residual stream, researchers can decompose the final output as a sum of contributions from every layer independently. This is how mechanistic interpretability studies which layers store which knowledge — each attention head and FFN layer has a cleanly separable contribution to the final output.

The residual stream grows — that's fine

Because every layer adds to x but nothing removes anything, the scale of x can gradually grow through the network. This is why LayerNorm (or RMSNorm in modern LLMs) is applied before each sub-layer in pre-norm transformers — it keeps the input to attention and FFN well-scaled even as the residual stream accumulates. The two mechanisms work together: residuals keep gradients healthy, normalisation keeps activations stable.

Residual connection
x = x + F(x) instead of x = F(x). The original signal is preserved and the sub-layer only computes a correction.
Identity shortcut
Another name for the residual path — the original x bypasses the sub-layer entirely and is added back at the end.
Residual stream
The running x that flows through the whole transformer. Every layer reads from it and adds to it.
Vanishing gradient
When gradients shrink to near-zero through many layers, early layers can't learn. Residuals solve this by providing a gradient highway.
Gradient highway
The direct backward path through the addition x + F(x). The gradient of a sum includes a full-strength copy through the identity branch.
ResNet (2015)
The paper "Deep Residual Learning for Image Recognition" (He et al.) introduced residual connections, enabling 152-layer networks and winning ImageNet. Transformers inherited this idea directly.

In one sentence

A residual connection is x = x + F(x) instead of x = F(x) — it lets the original signal bypass each sub-layer and flow straight through the network, giving gradients a free highway backward so early layers learn just as effectively as late ones.
