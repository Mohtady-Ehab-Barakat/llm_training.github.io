Regularisation & Dropout

Overfitting · L1 · L2 · weight decay · dropout · where each appears in transformers

The problem — overfitting

A model overfits when it learns the training data too well — including its noise and random fluctuations — and loses the ability to generalise to new data. Fitting a high-degree polynomial to noisy data gives a curve that passes through every training point perfectly but performs terribly on any new point, because it has memorised noise rather than learned the pattern.

Regularisation is any technique that discourages overfitting by adding a constraint or penalty that pushes the model toward simpler, more generalisable solutions.

L2 regularisation — weight decay

L2 adds a penalty to the loss proportional to the square of every weight:

Loss_total = Loss_task + λ · Σ w²

# Gradient of penalty w.r.t. w = 2λw
# Update becomes:
w ← w · (1 − 2·lr·λ) − lr · task_gradient
        ↑ this is the "decay" — w shrinks every step

Every weight gets multiplied by a number slightly less than 1 at every step. Weights not being pushed by data gradually shrink toward zero. This prevents any single weight from growing too large and creating wild behaviour. All weights are shrunk but none reach exactly zero.

Weight decay vs L2 in Adam: in standard SGD they are identical. In Adam, L2 regularisation (adding λw to the gradient before the adaptive step) and weight decay (multiplying weights directly by (1−λ) after the step) produce different effective penalties per weight — because the adaptive scaling distorts the L2 penalty. AdamW decouples them correctly. In transformers, always use weight decay (AdamW-style). Typical λ = 0.1.

L1 regularisation — Lasso

L1 adds a penalty proportional to the absolute value of every weight:

Loss_total = Loss_task + λ · Σ |w|

# Gradient of |w| = sign(w) — always ±1
# Even tiny weights feel constant pressure toward zero
# Many weights get pushed to EXACTLY zero → sparse model

L1 produces sparse models where many weights are exactly 0 — useful for automatic feature selection. Less common in deep learning because the non-smooth gradient can cause instability. L2 is the standard for neural networks.

Dropout — the most important regularisation in transformers

Dropout (Srivastava et al., 2014) is completely different from L1/L2. Instead of penalising weight magnitudes, it randomly deletes neurons during each forward pass of training.

Every neuron is independently zeroed with probability p (the drop rate). The surviving neurons are scaled up by 1/(1−p) to keep the expected sum of activations unchanged. At inference, all neurons are active and no scaling is needed.

Why dropout prevents overfitting

Ensemble perspective

Every forward pass uses a different random subset of neurons — a different "thinned" network. With n neurons and drop rate p, there are exponentially many possible masks. Each mask is effectively a different model. Dropout trains all of them simultaneously with shared parameters. At inference, the full network acts like an ensemble average of all these models. Ensembles always generalise better than single models.

Co-adaptation perspective

In an overfit network, neurons become dependent on each other — neuron A always fires with neuron B, developing a co-adapted pattern that memorises training data. Dropout breaks this: neuron A can never assume B will be present. Each neuron must learn something independently useful, forcing more robust, distributed representations.

Noise injection perspective

Dropout injects multiplicative noise into the activations. Training under noise is a classical regularisation technique — it prevents the model from fitting the fine-grained details of the training set. The random mask is different each batch, so no specific noise pattern can be memorised.

The inverted dropout formula

# Training (inverted dropout — the modern standard):
mask   = Bernoulli(1 − p)           # 1=keep, 0=drop
output = (mask * x) / (1 − p)      # scale up survivors

# Inference:
output = x                          # no masking, no scaling needed

# Why "inverted"? The scaling is done at training time.
# This means the inference code path is unchanged — just
# disable dropout and run normally. model.eval() in PyTorch.

Where dropout appears in a transformer

# After embedding:
x = dropout(token_emb + pos_emb, p=0.1)

# Per block — attention:
w   = dropout(softmax(scores), p=0.1)   # attention weights
out = dropout(Attn(LN(x)),     p=0.1)   # attention output
x   = x + out

# Per block — FFN:
h   = dropout(GELU(x @ W1),   p=0.1)   # after activation
out = dropout(h @ W2,          p=0.1)   # FFN output
x   = x + out

Four dropout applications per block, each with an independent random mask. Transformers use relatively low p = 0.1 because they are already regularised by weight decay, large datasets, and the structure of attention itself. During fine-tuning on small datasets, p = 0.2–0.3 is common to prevent overfitting the small fine-tuning set.

Training vs inference — the crucial difference

Dropout only runs during training. At inference all neurons are active and the network runs at full capacity. In PyTorch:

model.train()   # enables dropout, BatchNorm uses batch stats
model.eval()    # disables dropout, BatchNorm uses running stats

# Forgetting model.eval() before inference is a classic bug:
# the model still randomly zeros neurons → different output
# every run, even for identical inputs.

Dropout rates in practice

p = 0.5
Very aggressive. Classic for fully-connected layers in older models (AlexNet era). Too strong for transformers.
p = 0.1
Standard for transformer attention, FFN, and embedding layers. Used in GPT-2, BERT, LLaMA.
p = 0.0
No dropout. Sometimes used when the dataset is very large and regularisation isn't needed (some GPT-3 ablations).
Fine-tuning
Often increased to p = 0.2–0.3 when fine-tuning a large model on a small dataset.

Other regularisation techniques in LLMs

gradient clipping
Caps the gradient norm before each update (threshold = 1.0). Prevents a single bad batch from causing an enormous weight update that derails training. Universal in transformer training.
early stopping
Stop training when validation loss stops improving. Simple and effective. Requires a held-out validation set.
label smoothing
Instead of hard 0/1 targets, train on smooth targets (e.g. 0.9 for correct, 0.1/V spread over others). Prevents overconfidence. Used in the original transformer and some LLMs.
weight decay
L2 applied via AdamW. λ = 0.1 is the standard for GPT-3, LLaMA, and most modern LLMs. Applied to all parameters except LayerNorm γ/β and biases.
The practical stack in modern LLMs: weight decay (AdamW, λ=0.1) + dropout (p=0.1) + gradient clipping (norm=1.0). All three run simultaneously during training. At inference, only weight decay has any lasting effect — it shaped the weights. Dropout and gradient clipping play no role once training is done.
