The full flow — narrative

Input stage

Raw text is first split into tokens (subwords) and each is mapped to an integer ID. The token embedding table converts these IDs into D-dimensional float vectors — a simple lookup, no math. A positional encoding is then added element-wise to inject order, because attention has no built-in notion of sequence position.

Inside each transformer block (repeated N times)

Multi-head self-attention lets every token look at every other token. Each token produces a Query (what I need), Key (what I have), and Value (what I contribute). Dot products between Qs and Ks produce attention scores; softmax normalises them; multiplying by Vs blends the information. This runs H times in parallel with different learned projections — each head learns a different type of relationship. A causal mask sets future positions to −∞ so training on language modelling can't look ahead.

The first Add & Norm adds the original input back (residual connection) then normalises each token's vector to zero-mean unit-variance (LayerNorm). The residual is why 100-layer networks can be trained — gradients flow straight back without vanishing.

The feed forward network operates on each token independently (no cross-token mixing here). Two linear layers with a GELU activation expand to 4D then contract back to D. This is where factual knowledge is believed to be stored — each FFN layer acts like a differentiable key-value memory. The second Add & Norm wraps the FFN with the same residual + normalisation pattern.

Output stage

The linear (LM head) projects from D up to vocabulary size V, producing one raw score (logit) per possible next token. Most models share this weight matrix with the embedding table — it saves D×V parameters and works because both encode the same semantics. Softmax converts the logits into probabilities. During training, cross-entropy loss pushes up the probability of the correct next token. During inference, you sample from this distribution — temperature τ controls how flat or peaked it is.

Pre-norm vs post-norm: The original "Attention is All You Need" paper applied LayerNorm after the residual addition (post-norm). Modern LLMs (LLaMA, Mistral, GPT-4) apply it before the sub-layer (pre-norm). Pre-norm trains more stably at large scale and doesn't need learning rate warm-up tricks.

Shapes through the whole network

For a GPT-2 sized model: B = batch size, T = sequence length (1024), D = 768, H = 12 heads, V = 50,257, N = 12 layers.

Input
[B, T] — integer token IDs
After embed + pos
[B, T, D] — float vectors, this shape persists through all N blocks
Inside attn (QKV)
[B, H, T, D/H] — split across heads
Attn scores
[B, H, T, T] — every token attending to every token
After LM head
[B, T, V] — one score per vocab token, per position
After softmax
[B, T, V] — probabilities summing to 1 at each position

Where the parameters live

Embedding
V × D ≈ 38M (GPT-2). Or 0 if weight-tied with LM head.
Attention
4 × D² per block (Wq, Wk, Wv, Wo). 12 blocks × 4 × 768² ≈ 28M.
FFN
8 × D² per block (W1: D→4D, W2: 4D→D). 12 blocks × 8 × 768² ≈ 57M.
LayerNorm
4 × D per block (γ and β, twice). Tiny.
LM head
D × V ≈ 38M. Usually weight-tied → free.
