What is normalization?

During a forward pass, the numbers flowing through the network (activations) can drift into bad ranges — exploding to huge values, collapsing to near-zero, or accumulating a large mean offset. Any of these makes training unstable or slow. Normalization is a forced housekeeping step: rescale every vector so it has mean ≈ 0 and standard deviation ≈ 1 before passing it to the next sub-layer.

The formula for LayerNorm applied to a single vector x of length D:

μ = mean(x)                     # scalar mean over D dimensions
σ = std(x)                      # scalar std  over D dimensions
x̂ = (x − μ) / (σ + ε)          # ε ≈ 1e-5 avoids div-by-zero

output = γ · x̂ + β             # γ, β learned per-dimension

The learned γ (scale) and β (shift) are critical — without them you'd permanently force every vector into the same shape, destroying the network's expressiveness. With them the network can learn "I want dimension 42 to live around mean 3, std 0.5" — but the default at init is normalized (γ=1, β=0), and it only deviates when gradients push it to.

LayerNorm vs BatchNorm — why transformers need LayerNorm

BatchNorm normalises across the batch dimension — it computes mean and std across all examples in the batch for each feature. This works for CNNs but has a fatal flaw for transformers: during inference with batch size 1, you need running statistics tracked during training — a messy approximation. It also makes each example's normalisation depend on the rest of the batch, which breaks autoregressive generation.

LayerNorm normalises across the feature dimension D — it computes mean and std across all D dimensions for each token, independently. Batch size doesn't matter. Every token is normalised using only its own values. This is why every transformer uses LayerNorm.

Why pre-norm wins — gradient flow

The difference sounds small but has a huge effect during backpropagation. In post-norm, the gradient must pass through LayerNorm at every residual junction. LayerNorm divides by σ in the forward pass, which means multiplying by 1/σ in the backward pass — this can amplify or shrink the gradient unpredictably, especially early in training when σ is unstable.

In pre-norm, the residual addition is a raw x + F(LN(x)). The gradient of a sum is just the sum of the gradients — so the gradient flows directly back through the addition, bypassing any norm. The residual highway is perfectly clean. Drag the sliders to see the difference.

RMSNorm — what LLaMA and Mistral use

LLaMA, Mistral, Gemma and most recent models use RMSNorm instead of full LayerNorm. It drops the mean-centering step entirely:

RMS(x) = sqrt( mean(x²) )      # no mean subtraction
x̂ = x / RMS(x)                 # just scale
output = γ · x̂                 # only γ — no β shift

The intuition: mean-centering contributes relatively little to training stability — the scale normalisation does most of the work. Dropping it removes one operation and one learned parameter per dimension. RMSNorm is roughly 15% faster than LayerNorm and achieves equal training stability. It is always used in pre-norm position.

In practice: Every major LLM released after 2020 uses pre-norm + RMSNorm. The combination is so dominant that "pre-norm" and "RMSNorm" are often mentioned together. LLaMA 1/2/3, Mistral, Gemma, Falcon, and most others all follow this pattern.
