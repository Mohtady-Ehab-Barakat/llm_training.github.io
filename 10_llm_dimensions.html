Every dimension explained

V — Vocabulary size

also: vocab_size

Before any neural network runs, text is split into subword tokens and each is mapped to an integer ID from 0 to V−1. V is how many distinct tokens exist. GPT-2: V = 50,257. LLaMA: V = 32,000. Larger vocabulary means fewer tokens per sentence (more efficient), but a bigger embedding table and LM head.

Every forward pass starts as integers in [0, V) and ends as a probability distribution over V options. V directly sets the size of the two most expensive weight matrices: the embedding table (V × D) and the LM head (D × V).

D — Model dimension

also: d_model · hidden_size · embedding_dim · n_embd

The single most important number. Every token is represented as a vector of D floating-point numbers as it flows through the entire network. D is the width of the residual stream. Every sub-layer reads D numbers in and writes D numbers out — the shape [B, T, D] never changes across all L blocks. Only the content changes as information accumulates.

GPT-2 Small: D = 768. GPT-3: D = 12,288. LLaMA-7B: D = 4,096. LLaMA-70B: D = 8,192.

Doubling D roughly quadruples parameters and FLOPs — every weight matrix is proportional to D² (attention has 4D², FFN has ~8D²). D is the primary lever for scaling model capacity.

L — Number of layers

also: n_layers · num_hidden_layers · depth

How many transformer blocks are stacked sequentially. Each block contains one attention sub-layer and one FFN sub-layer, both wrapped in residual connections. The token representations flow through all L blocks one by one.

GPT-2 Small: L = 12. GPT-3: L = 96. LLaMA-7B: L = 32. LLaMA-70B: L = 80.

Increasing L adds parameters linearly. But depth also gives the model more sequential "reasoning steps" — a deeper model can do more multi-step processing even at the same parameter count as a wider but shallower model. Modern practice balances both. Early layers tend to handle syntax and local patterns; later layers handle semantics and long-range reasoning.

H — Number of attention heads

also: n_heads · num_attention_heads

Each transformer block runs self-attention H times in parallel, each time using a different learned projection of Q, K, V. Different heads specialise in different relationships — one might track syntactic subject-verb agreement, another coreference ("it" referring to an earlier noun), another positional proximity.

Key constraint: D must be divisible by H. The model dimension is split evenly across heads: head_dim = D / H. GPT-2 Small: H = 12, head_dim = 64. LLaMA-70B: H = 64, head_dim = 128.

Head dimension is almost always 64 or 128 in practice — these sizes work well with GPU tensor cores and have been empirically validated. You rarely want head_dim below 32 or above 256.

head_dim — Head dimension

also: d_head · head_size | derived: head_dim = D / H

The dimension of the space each attention head works in. Inside each head, Q and K are head_dim-dimensional vectors. The attention score between two tokens is their dot product divided by √head_dim. This scaling is critical — without it, dot products grow large, softmax saturates (all attention goes to one token), and gradients vanish.

After all H heads compute their weighted sums, results are concatenated back to D dimensions and projected through Wₒ. The full attention path is: D → split into H × head_dim → attend per head → concat → project → D.

d_ffn — FFN hidden dimension

also: intermediate_size · ffn_dim · d_inner | typically 4×D or 8/3×D

Inside the feed-forward network, each token vector expands from D to d_ffn (gaining compute depth), then contracts back to D. Classic transformers use 4×D (GPT-2: 768 → 3,072 → 768). LLaMA uses SwiGLU with 8/3×D to keep the parameter count equal despite using two projection matrices.

The FFN is where the model stores factual "knowledge" — think of each FFN layer as a key-value memory bank. Wider FFN = more storage capacity. The FFN contains roughly two-thirds of all parameters in a standard transformer (attention has 4D², FFN has ~8D² with a 4× multiplier).

T — Context length (sequence length)

also: seq_len · max_position_embeddings · context_window

How many tokens the model processes at once. This is the length of the sequence dimension in every tensor. Attention computes a T×T score matrix — every token attending to every other token. This is why attention is O(T²) in compute and memory — doubling context quadruples the attention cost.

GPT-2: T = 1,024. GPT-3: T = 2,048. LLaMA-2: T = 4,096. LLaMA-3: T = 128,000 (with RoPE scaling).

Context length determines how much text the model can "see" at once, the size of the [B, H, T, T] attention score matrix, and the size of the KV cache during inference.

KV cache

size = 2 × L × T × D × bytes_per_value

During generation, the model produces one token at a time. Without caching, you'd recompute K and V for all previous tokens at every step — wasteful. The KV cache stores the K and V tensors for every already-generated token, for every layer. Each new token only needs to compute its own K and V.

For LLaMA-7B at T = 4,096 in float16: 2 × 32 × 4,096 × 4,096 × 2 bytes ≈ 2 GB — just for the cache, on top of the ~14 GB model weights. Long contexts are expensive because the cache grows linearly with sequence length and must fit in GPU memory alongside the model.

B — Batch size

always the first dimension: [B, T, D]

How many independent sequences are processed simultaneously in one forward pass. During training: B = 32 to 4,096+ (larger batches = more stable gradients, better GPU utilisation, but more memory). During single-user inference: B = 1. The batch dimension is always first in every tensor and never affects the model's logic — it's just parallelism.

Where every parameter lives

── Input/output ─────────────────────────────────────── Embedding table: V × D Positional embeddings: T × D (if learned; 0 for RoPE) LM head: D × V (often weight-tied → 0 extra) ── Per transformer block (× L) ──────────────────────── Attention Wq: D × D Attention Wk: D × D Attention Wv: D × D Attention Wo: D × D total: 4 × D² FFN W1: D × d_ffn FFN W2: d_ffn × D total: 2 × D × d_ffn LayerNorm (×2): 4 × D (γ and β, twice per block) ── Total ─────────────────────────────────────────────── Total = V×D + L×(4D² + 2×D×d_ffn + 4D) + D×V

The full shape flow

Tracing a single sequence through a model with D=4096, H=32, L=32, V=32000, T=4096:

Input:      [B, T]              = [1, 4096]   integer token IDs

Embed:      [B, T, D]           = [1, 4096, 4096]
+ pos enc:  [B, T, D]           = [1, 4096, 4096]   shape unchanged

Per block (× 32):
  Q,K,V:    [B, H, T, head_dim] = [1, 32, 4096, 128]
  Scores:   [B, H, T, T]        = [1, 32, 4096, 4096]  ← O(T²) cost
  Attn out: [B, T, D]           = [1, 4096, 4096]
  FFN mid:  [B, T, d_ffn]       = [1, 4096, 10922]
  FFN out:  [B, T, D]           = [1, 4096, 4096]

LM head:    [B, T, V]           = [1, 4096, 32000]
Softmax:    [B, T, V]           = [1, 4096, 32000]   probabilities
The key insight: the shape [B, T, D] is the skeleton of the whole transformer — it never changes across all L blocks. Only the content of those D numbers changes as each block enriches the representations. The residual stream is the river; each block is a tributary adding information to it.

head_dim rule
Almost always 64 or 128. Chosen to be hardware-friendly and empirically stable.
FFN ratio
4× for classic transformers (GPT). 8/3× for SwiGLU models (LLaMA). Both give similar parameter counts per block.
GQA
Grouped Query Attention — used in LLaMA-2-70B and Mistral. Fewer K,V heads than Q heads (e.g. 8 KV heads for 64 Q heads), shrinking the KV cache dramatically with minimal quality loss.
