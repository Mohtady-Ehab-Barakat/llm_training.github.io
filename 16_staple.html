STAPLE

Simultaneous Truth and Performance Level Estimation — combining noisy annotators without ground truth

Sensitivity & specificity — from the beginning

Before STAPLE makes sense, you need a firm grip on sensitivity and specificity. They describe the two independent ways an annotator (or any binary classifier) can be right or wrong.

Every item has a true label (positive or negative) and an annotator assigns a predicted label (1 or 0). This produces four possible outcomes:

Annotator says
1 (positive) 0 (negative)
True label 1 (positive) True Positive (TP)
Correctly found		 False Negative (FN)
Missed it
0 (negative) False Positive (FP)
False alarm		 True Negative (TN)
Correctly absent

Sensitivity (p) — also called recall or true positive rate — answers: "Given that the true label IS positive, how often does the annotator say positive?"

sensitivity p = TP / (TP + FN) = P(annotate=1 | truth=1)

p = 1.0 → never misses a positive (perfect recall)
p = 0.5 → misses half of all positives (random)
p = 0.0 → always labels positives as negative (worst)

Specificity (q) — also called true negative rate — answers: "Given that the true label IS negative, how often does the annotator say negative?"

specificity q = TN / (FP + TN) = P(annotate=0 | truth=0)

q = 1.0 → never raises a false alarm (perfect at negatives)
q = 0.5 → randomly labels half of negatives as positive
q = 0.0 → always labels negatives as positive (worst)
Critical point: sensitivity and specificity are conditional probabilities — they are defined relative to the true label. They describe the annotator's behaviour, not the item's probability of being positive. An annotator can have high sensitivity and low specificity (a "yes-sayer" who labels everything positive) or vice versa.

The multi-annotator problem

You have N items to label. You hire J annotators. They disagree. Who is right?

The naive answer is majority vote — take whatever most annotators say. But this treats every annotator as equally reliable. An expert who agrees with one careless annotator gets outvoted by three careless annotators. You're counting votes, not weighting by quality.

STAPLE (Warfield et al., 2004) solves this by simultaneously estimating the hidden true labels and each annotator's reliability from the annotations alone — no gold standard needed. It uses sensitivity and specificity to model each annotator, and an EM algorithm to fit the model.

The STAPLE model

For each annotator j, STAPLE models exactly two numbers:

p_j = sensitivity = P(annotator j says 1 | true label is 1)
q_j = specificity = P(annotator j says 0 | true label is 0)

The true label T_i for item i is hidden (that's what we want to find). STAPLE also estimates the prevalence f = P(T_i = 1). Given the true label, each annotator's annotation is conditionally independent of the others. So the probability of observing all annotations for item i is:

P(d_i | T_i=1) = Π_j  p_j^d_ij  ×  (1-p_j)^(1-d_ij)
P(d_i | T_i=0) = Π_j  (1-q_j)^d_ij  ×  q_j^(1-d_ij)

# d_ij = annotation of item i by annotator j (0 or 1)
# If T_i=1: annotator j's probability of saying 1 is p_j
# If T_i=0: annotator j's probability of saying 0 is q_j

The EM algorithm — how STAPLE fits the model

STAPLE uses Expectation-Maximization (EM). The hidden variable is T_i (the true label). EM alternates between two steps:

# ── E-step: compute posterior probability of true label ──────
W_i = P(T_i=1 | all annotations, current p, q, f)

    = [f × Π_j p_j^d_ij(1-p_j)^(1-d_ij)]  /
      [f × Π_j p_j^d_ij(1-p_j)^(1-d_ij)   +
       (1-f) × Π_j (1-q_j)^d_ij q_j^(1-d_ij)]

# W_i ≈ 1 → probably positive
# W_i ≈ 0 → probably negative
# W_i ≈ 0.5 → uncertain

# ── M-step: update annotator quality estimates ───────────────
p_j = Σ_i W_i × d_ij  /  Σ_i W_i
    # fraction of likely-positive items that annotator j labelled 1

q_j = Σ_i (1-W_i) × (1-d_ij)  /  Σ_i (1-W_i)
    # fraction of likely-negative items that annotator j labelled 0

f   = Σ_i W_i / N
    # prevalence estimate

# ── Repeat until W_i and parameters stop changing ───────────
# Initialise: W_i = majority vote, p_j = q_j = 0.8

EM convergence

Watch how the estimated sensitivity (p) and specificity (q) for each annotator converge over EM iterations. The algorithm bootstraps from majority vote — initially everyone looks similar, then the reliable and unreliable annotators separate.

p (sensitivity) and q (specificity) converging over 60 iterations
Sensitivity p_j — P(says 1 | truth=1)
Specificity q_j — P(says 0 | truth=0)
Convergence story: all annotators start with the same estimated p=q=0.8. After just a few iterations, the expert separates from the poor annotator. The EM is self-reinforcing: annotators who agree with the current truth estimate get higher p/q, which makes their future votes count more, which makes the truth estimates more accurate. The algorithm bootstraps quality from disagreement patterns alone.

What STAPLE outputs

  • W_i for every item: a posterior probability in [0, 1] that the true label is 1. Threshold at 0.5 for a hard label, or keep as a soft label for training.
  • p_j and q_j for every annotator: their estimated sensitivity and specificity. Use these to rank annotators, flag low-quality ones, or exclude them.
  • f: estimated prevalence of positive items in the dataset.

Extensions

  • Multi-class STAPLE: replace sensitivity/specificity with a K×K confusion matrix per annotator — P(annotator says k' | truth is k).
  • Spatial STAPLE: for medical image segmentation, add a spatial prior (Markov Random Field) so neighbouring voxels inform each other's W estimate.
  • Continuous STAPLE: for real-valued ratings (1–5 stars), model each annotator with a bias and variance instead of p and q.

In the LLM context

  • RLHF preference data: multiple human raters compare model outputs (A vs B). STAPLE recovers which rater is reliable and produces cleaner preference labels for the reward model.
  • Instruction quality annotation: multiple raters score instruction-following quality. STAPLE filters noisy labels.
  • Safety annotation: multiple annotators flag harmful content. Annotator reliability varies widely; STAPLE automatically down-weights noisy flaggers.
  • LLM-as-annotator ensembles: run the same labeling prompt through multiple LLMs or temperature settings. Treat each LLM as an "annotator" and run STAPLE — often outperforms simple majority vote of LLM outputs.
Sensitivity p
P(annotate=1 | truth=1). How often an annotator correctly identifies positives. High p = rarely misses real positives.
Specificity q
P(annotate=0 | truth=0). How often an annotator correctly identifies negatives. High q = rarely raises false alarms.
W_i
STAPLE's posterior estimate P(T_i=1 | all annotations). The soft true label for item i. The main output of STAPLE.
EM
Expectation-Maximization. Alternates between estimating hidden variables (W_i) given current params, and updating params given current W_i. Guaranteed to increase likelihood at every step.
Conditional independence
STAPLE assumes annotators' errors are independent given the true label. Knowing one annotator made an error tells you nothing about another's error.
Prevalence f
Estimated fraction of items with true label 1. Also learned by STAPLE from the data.
Yes-sayer / No-sayer
An annotator with high p but low q labels almost everything positive. High q but low p labels almost everything negative. STAPLE identifies both patterns.
